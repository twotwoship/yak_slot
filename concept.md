# 🛡️ SLOT-GUARD 시스템 구조 및 사용 가이드

본 문서는 **SLOT-GUARD** 프로젝트의 장치별 역할 분담과 실제 관리자의 사용 프로세스를 정의합니다.

---

## 1. 장치별 역할 확정

> ⚠️ **핵심 개발 원칙**
> * 모터와 센서의 **실시간 제어**는 **ATmega128A**가 전담합니다.
> * **Raspberry Pi**는 제어에 관여하지 않고, 데이터 및 웹 서버 등 **관리 기능만 담당**합니다.

| 장치 / 모듈 | 담당 기능 | 구현 수단 |
| :--- | :--- | :--- |
| **ATmega128A** | 모터·센서·복용 허용시간 제어 | C 펌웨어 |
| **DS1307** | 인터넷 연결 없이 현재 시각 유지 | $I^2C$ 통신 |
| **Raspberry Pi** | 일정 저장, 로그 저장, 스마트폰 화면 제공 | Python |
| **Raspberry Pi Wi-Fi** | 스마트폰 접속용 자체 AP 구축 | NetworkManager |
| **Raspberry Pi 웹서버** | 일정 등록 · 로그 조회 시스템 | Flask |
| **Raspberry Pi DB** | 일정 및 배출 이력 보관 | SQLite |
| **Raspberry Pi ↔ ATmega** | 명령 및 상태 데이터 전송 | UART (pySerial) |
| **스마트폰** | 관리자 입력 및 조회 화면 | 기본 웹 브라우저 |

---

## 2. 실제 사용 과정 (워크플로우)

관리자(담당자)가 **1~2주에 한 번 현장에 방문**하여 시스템을 점검하고 일정을 갱신할 때의 서순입니다. 본 시스템은 외부 인터넷 연결 없이 **라즈베리파이 내부 웹페이지를 Wi-Fi로 직접 받아 구동**됩니다.

### 📱 접속 및 관리 순서
1. **스마트폰 Wi-Fi 설정**을 켭니다.
2. Wi-Fi 목록에서 **`SLOT-GUARD`** 네트워크를 선택하여 연결합니다.
3. 설정된 비밀번호를 입력합니다.
4. *"인터넷 연결이 확실치 않다"*는 안내 문구가 뜨더라도 **`연결 유지`**를 선택합니다.
5. 장치 외관에 부착된 **QR 코드를 촬영**합니다.
6. 스마트폰 기본 브라우저를 통해 관리 화면에 접속합니다.
7. 시스템의 지난 기록(배출 이력)을 확인하고 다음 복용 일정을 등록합니다.

### 🌐 접속 정보
* **QR 코드 내장 URL:** `http://192.168.4.1:5000`
